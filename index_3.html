GitHub Auto-Update
Set up automatic daily dashboard updates via GitHub Actions — so the CAPAD dashboard rebuilds itself every morning without any manual file uploads.
How this works: GitHub Actions will run a scheduled workflow every weekday morning. It reads your stored CAPAD data (via this form), generates updated JavaScript arrays, and commits the new capad_dashboard_1.html to your repo — which GitHub Pages then serves automatically. One-time setup takes about 10 minutes.

Step 1 — Create a GitHub Personal Access Token

The workflow needs permission to write to your repository.

1

Go to GitHub Settings → Developer Settings → Personal Access Tokens

Navigate to github.com → Settings → Developer Settings → Personal Access Tokens → Tokens (classic)

Open GitHub Token Page →
2

Generate token with repo scope

Name it CAPAD_DEPLOY, set expiration to 1 year, and tick the repo checkbox. Click "Generate token" and copy the token immediately — you won't see it again.

3

Add token as a repository secret

Go to github.com/MalanOlinsky/DailyCapad → Settings → Secrets and variables → Actions → New repository secret

Name: CAPAD_TOKEN  ·  Value: paste your token

Step 2 — Create the GitHub Actions Workflow File

Create this file in your repository at the exact path shown. Copy the YAML below, then commit it.

1

Create file at: .github/workflows/capad-update.yml

In your repo on GitHub: click Add file → Create new file, type the path above as the filename, paste the content below.

name: CAPAD Daily Update on: schedule: - cron: '30 5 * * 1-5' # 07:30 SAST (05:30 UTC) Mon-Fri workflow_dispatch: # Allow manual trigger from Actions tab jobs: update-capad: runs-on: ubuntu-latest steps: - name: Checkout repository uses: actions/checkout@v4 with: token: ${{ secrets.CAPAD_TOKEN }} - name: Setup Node.js uses: actions/setup-node@v4 with: node-version: '20' - name: Install dependencies run: npm install node-fetch@2 - name: Generate updated CAPAD dashboard run: node .github/scripts/update-capad.js env: GITHUB_TOKEN: ${{ secrets.CAPAD_TOKEN }} - name: Commit and push changes run: | git config user.name "CAPAD Bot" git config user.email "capad-bot@avior.co.za" git add capad_dashboard_1.html git diff --staged --quiet || git commit -m "CAPAD auto-update $(date +'%Y-%m-%d')" git push

Step 3 — Create the Update Script

Create this file at .github/scripts/update-capad.js in your repo. This script reads your stored CAPAD data and rebuilds the dashboard HTML.

// .github/scripts/update-capad.js // Reads CAPAD data from storage API and rebuilds capad_dashboard_1.html const fs = require('fs'); // ── Read stored CAPAD data ────────────────────────────────────────────────── // The form on index_3.html stores data in localStorage and syncs from capad_data.json. // For GitHub Actions, data is passed via environment variable or a JSON file // that you export from the form and commit to the repo. // // OPTION A (recommended): Export JSON from the form and commit it daily. // 1. On the CAPAD History page, click "Export CSV" then manually save // a capad_data.json file to your repo once (or automate via the form). // 2. This script reads that JSON file. // // OPTION B: Use a cloud key-value store (e.g. Upstash Redis, free tier) // and have the form POST data there directly. const DATA_FILE = 'capad_data.json'; let records = []; try { records = JSON.parse(fs.readFileSync(DATA_FILE, 'utf8')); console.log(`Loaded ${records.length} CAPAD records from ${DATA_FILE}`); } catch(e) { console.error('Could not read capad_data.json:', e.message); process.exit(0); // Exit cleanly — don't overwrite dashboard with empty data } if (records.length === 0) { console.log('No records found — skipping update'); process.exit(0); } // ── Build JS arrays ───────────────────────────────────────────────────────── const R = records; const N = R.length; const arr = (name, vals) => `const ${name}=[${vals.join(',')}];`; const latest = R[R.length-1]; const prev = R.length > 1 ? R[R.length-2] : latest; const jsBlock = [ `const DAYS=[${R.map(r=>`'${r.dateLabel}'`).join(',')}];`, `const DATES_FULL=[${R.map(r=>`'${r.date}'`).join(',')}];`, `const N=${N};`, arr('ALC', R.map(r=>r.alc)), arr('TRR', R.map(r=>r.trr)), 'const SURPLUS=ALC.map((v,i)=>v-TRR[i]);', 'const COVERAGE=ALC.map((v,i)=>v/TRR[i]*100);', arr('TRADING_LIMIT', R.map(r=>Math.round(r.tradingLimit))), arr('CAPITAL_RES', R.map(r=>r.capitalRes)), `const IMPAIRED=Array(N).fill(${R[0].impaired});`, arr('ILLIQUID', R.map(r=>r.illiquid)), arr('UNREALISED', R.map(r=>r.unrealised)), arr('INCOME', R.map(r=>r.income)), arr('EXPENSE', R.map(r=>r.expense)), 'const UNAUD_PNL=INCOME.map((v,i)=>v-EXPENSE[i]);', 'const PNL_DELTA=UNAUD_PNL.map((v,i)=>i===0?0:v-UNAUD_PNL[i-1]);', 'const MARGIN=INCOME.map((v,i)=>UNAUD_PNL[i]/v*100);', `const BASE_REQ=Array(N).fill(${R[0].baseReq});`, arr('LER', R.map(r=>r.ler||0)), arr('POS_RISK', R.map(r=>r.posRisk)), arr('CP_NON', R.map(r=>r.cpNon)), arr('CP_LOANS', R.map(r=>r.cpLoans)), arr('CP_CTRL', R.map(r=>r.cpCtrl||0)), `const SHAREHOLDERS=Array(N).fill(${R[0].shareholders});`, `const RETAINED=Array(N).fill(${R[0].retained});`, `const TREASURY=Array(N).fill(${R[0].treasury});`, `const DIVIDENDS=Array(N).fill(0);`, `const SUB_LOANS=Array(N).fill(${R[0].subloans});`, ].join('\n'); // ── Read and patch capad_dashboard_1.html ──────────────────────────────────── let html = fs.readFileSync('capad_dashboard_1.html', 'utf8'); // Replace the data block between DATA_START and DATA_END markers const START_MARKER = '// DATA_START'; const END_MARKER = '// DATA_END'; const startIdx = html.indexOf(START_MARKER); const endIdx = html.indexOf(END_MARKER); if (startIdx === -1 || endIdx === -1) { console.error('Could not find DATA_START / DATA_END markers in capad_dashboard_1.html'); console.log('Add these comments around the const DAYS/ALC/TRR etc. block in the script section'); process.exit(1); } const updated = html.slice(0, startIdx + START_MARKER.length) + '\n' + jsBlock + '\n' + html.slice(endIdx); // Update header meta const dayCount = N; const latestDate = latest.date; const patchedHtml = updated .replace(/\d+ trading days/, `${dayCount} trading days`) .replace(/Updated: \d+ \w+ \d+/, `Updated: ${new Date().toLocaleDateString('en-GB',{day:'numeric',month:'long',year:'numeric'})}`); fs.writeFileSync('capad_dashboard_1.html', patchedHtml); console.log(`Dashboard updated: ${dayCount} days, latest ${latestDate}, ALC R${(latest.alc/1e6).toFixed(1)}M`);

Step 4 — Add DATA_START / DATA_END markers to the CAPAD dashboard

Open capad_dashboard_1.html and find the line that starts with const DAYS=[...]. Add these two comment markers around the data block so the script knows what to replace:

// DATA_START const DAYS=[...]; const DATES_FULL=[...]; const N=34; const ALC=[...]; ... (all the other const arrays) ... const SUB_LOANS=Array(N).fill(128122438); // DATA_END

Only the two comment lines need to be added — the data arrays between them will be replaced automatically on each run.

Step 5 — Export data file and commit it

The GitHub Actions script reads a capad_data.json file from your repo. Export it from the History page and commit it once to get started. After that, use the button below each time you add a new entry.

1

Export JSON from the History page

Go to CAPAD History → click "Copy JS Arrays" — this gives you the current data. To get raw JSON, use the button below.

2

Commit capad_data.json to your repo root

Upload the file to github.com/MalanOlinsky/DailyCapad in the repo root. Every time you save a new CAPAD entry, re-export and re-commit this file — or the Actions workflow won't have the latest data.

3

Trigger the workflow manually to test

Go to github.com/MalanOlinsky/DailyCapad → Actions → CAPAD Daily Update → Run workflow. Check the run log — if successful, your dashboard will update within 30 seconds.
